# 1. HTML - Скелет веб-страницы

## 1.1 Что такое HTML простыми словами

**HTML** (HyperText Markup Language) = язык разметки для веб-страниц

### Аналогия

```
HTML = скелет человека
CSS  = одежда и внешний вид
JS   = мышцы и движения
```

HTML отвечает за **структуру** страницы:
- Где находится заголовок
- Где текст
- Где картинки
- Где кнопки

### Как это выглядит

```html
<h1>Привет, мир!</h1>
<p>Это абзац текста.</p>
<button>Нажми меня</button>
```

**Браузер превращает это в:**

```
━━━━━━━━━━━━━━━━━━━
  Привет, мир!
━━━━━━━━━━━━━━━━━━━

Это абзац текста.

┌─────────────────┐
│  Нажми меня     │
└─────────────────┘
```

---

## 1.2 Структура HTML документа

Каждая веб-страница имеет базовую структуру:

```html
<!DOCTYPE html>
<html>
  <head>
    <title>Название страницы</title>
  </head>
  <body>
    <!-- Здесь весь контент страницы -->
  </body>
</html>
```

### Разбор частей

```
<!DOCTYPE html>          ← "Это HTML5 документ"
<html>                   ← Корневой элемент

  <head>                 ← "Голова" (невидимая часть)
    <title>...</title>   ← Заголовок вкладки браузера
    <meta>               ← Мета-информация
    <link>               ← Подключение CSS
  </head>

  <body>                 ← "Тело" (видимая часть)
    ... контент ...      ← Всё что видит пользователь
  </body>

</html>
```

**Важно:**
- `<head>` = настройки и подключения (не видно на странице)
- `<body>` = весь видимый контент

---

## 1.3 Основные теги

HTML состоит из **тегов** (tags). Тег = команда браузеру "сделай элемент".

### Заголовки

```html
<h1>Главный заголовок</h1>
<h2>Подзаголовок</h2>
<h3>Подподзаголовок</h3>
```

`h1` → `h6` (от большого к маленькому)

### Текст

```html
<p>Абзац текста</p>
<span>Кусочек текста</span>
<strong>Жирный текст</strong>
<em>Курсивный текст</em>
```

### Контейнеры

```html
<div>Блочный контейнер (занимает всю ширину)</div>
<section>Секция страницы</section>
<header>Шапка сайта</header>
<footer>Подвал сайта</footer>
```

**Визуализация:**

```
┌────────────────────────────┐
│ <header> Шапка сайта       │
├────────────────────────────┤
│ <section>                  │
│   <h1> Заголовок           │
│   <p> Текст                │
│ </section>                 │
├────────────────────────────┤
│ <footer> Подвал            │
└────────────────────────────┘
```

### Ссылки и изображения

```html
<a href="https://google.com">Ссылка на Google</a>
<img src="photo.jpg" alt="Описание картинки">
```

### Кнопки и формы

```html
<button>Нажми меня</button>
<input type="text" placeholder="Введите текст">
<textarea>Многострочное поле</textarea>
```

---

## 1.4 Атрибуты

Атрибуты = дополнительная информация для тега

### Синтаксис

```html
<тег атрибут="значение">Контент</тег>
```

### Основные атрибуты

```html
<!-- class - имя для стилизации -->
<div class="container">...</div>

<!-- id - уникальный идентификатор -->
<div id="header">...</div>

<!-- href - ссылка (для <a>) -->
<a href="https://example.com">Ссылка</a>

<!-- src - путь к файлу (для <img>) -->
<img src="image.jpg">

<!-- alt - альтернативный текст (для <img>) -->
<img src="photo.jpg" alt="Описание фото">
```

**Разница между `class` и `id`:**

```
class → может быть у многих элементов
id    → должен быть уникальным (один на странице)
```

# 2. CSS - Внешний вид страницы

## 2.1 Что такое CSS

**CSS** (Cascading Style Sheets) = язык стилей для HTML

### Что делает CSS

```
HTML без CSS:
┌────────────┐
│ Заголовок  │
│ Текст      │
│ [Кнопка]   │
└────────────┘

HTML + CSS:
╔══════════════════╗
║  🎨 Заголовок    ║ ← красивый шрифт
╠══════════════════╣
║ Текст с          ║ ← цвета
║ отступами        ║ ← размеры
║ ╭──────────╮     ║
║ │ Кнопка   │     ║ ← анимация
║ ╰──────────╯     ║
╚══════════════════╝
```

CSS отвечает за:
- 🎨 Цвета
- 📏 Размеры
- 📍 Расположение
- 🌈 Эффекты (тени, анимации)
- 📱 Адаптивность

---

## 2.2 Как подключить CSS к HTML

### Способ 1: Внешний файл (рекомендуется)

```html
<!-- В <head> -->
<link rel="stylesheet" href="styles.css">
```

### Способ 2: Внутри HTML

```html
<style>
  h1 { color: blue; }
</style>
```

### Способ 3: Inline (не рекомендуется)

```html
<h1 style="color: blue;">Заголовок</h1>
```

**Почему внешний файл лучше?**
- ✅ Один файл для всего сайта
- ✅ Легко менять дизайн
- ✅ Браузер кэширует файл (быстрее загрузка)

---

## 2.3 Селекторы

**Селектор** = способ выбрать элемент для стилизации

### По тегу

```css
p {
  color: black;
}
```

**Что делает:** Все `<p>` станут чёрными

### По классу

```css
.container {
  width: 1200px;
}
```

**Что делает:** Все элементы с `class="container"` будут 1200px

```html
<div class="container">...</div>
```

### По ID

```css
#header {
  background: blue;
}
```

**Что делает:** Элемент с `id="header"` получит синий фон

```html
<header id="header">...</header>
```

### Комбинации

```css
/* Все <p> внутри .container */
.container p {
  font-size: 16px;
}

/* Элемент с классом .btn и .primary одновременно */
.btn.primary {
  background: green;
}
```

---

## 2.4 Основные свойства

### Цвета

```css
.element {
  color: red;              /* Цвет текста */
  background-color: blue;  /* Цвет фона */
}
```

**Форматы цветов:**
```css
color: red;              /* Название */
color: #FF0000;          /* HEX */
color: rgb(255, 0, 0);   /* RGB */
```

### Размеры

```css
.element {
  width: 300px;    /* Ширина */
  height: 200px;   /* Высота */
  font-size: 16px; /* Размер шрифта */
}
```

### Отступы и границы

```css
.box {
  margin: 20px;     /* Внешний отступ */
  padding: 10px;    /* Внутренний отступ */
  border: 1px solid black; /* Рамка */
}
```

**Box Model:**

```
┌─────────────────────────────┐
│  margin (внешний отступ)    │
│  ┌───────────────────────┐  │
│  │ border (рамка)        │  │
│  │ ┌─────────────────┐   │  │
│  │ │ padding         │   │  │
│  │ │ ┌───────────┐   │   │  │
│  │ │ │  content  │   │   │  │
│  │ │ │           │   │   │  │
│  │ │ └───────────┘   │   │  │
│  │ └─────────────────┘   │  │
│  └───────────────────────┘  │
└─────────────────────────────┘
```

---

## 2.6 Адаптивность (основы)

**Адаптивность** = сайт хорошо выглядит на всех устройствах

### Media Queries

```css
/* Для экранов меньше 768px (мобильные) */
@media (max-width: 768px) {
  .container {
    width: 100%;
    padding: 10px;
  }
}
```

**Визуализация:**

```
DESKTOP (> 768px):
┌──────────────────────────────┐
│ [Logo] [Menu] [Menu] [Menu]  │
│                              │
│  Content in 3 columns        │
│  ┌────┐ ┌────┐ ┌────┐       │
│  │    │ │    │ │    │       │
└──────────────────────────────┘

MOBILE (< 768px):
┌────────────┐
│   [Logo]   │
│ [☰ Menu]   │
│            │
│  Content   │
│  ┌──────┐  │
│  │      │  │
│  └──────┘  │
│  ┌──────┐  │
│  │      │  │
│  └──────┘  │
└────────────┘
```

### Viewport Meta Tag

**Обязательно добавить в `<head>`:**

```html
<meta name="viewport" content="width=device-width, initial-scale=1.0">
```

**Без этого тега** мобильный браузер отобразит сайт как на десктопе (всё мелкое).

# 3. JavaScript - Минимум для понимания

## 3.1 Что такое JavaScript

**JavaScript** (JS) = язык программирования для веб-страниц

### Что делает JS

```
HTML + CSS = статичная страница (картинка)
HTML + CSS + JS = интерактивная страница (реагирует на действия)
```

**Примеры использования:**
- 🖱️ Клик по кнопке → открылось окно
- 📝 Валидация формы → "Введите email"
- 🔄 Динамическая загрузка контента
- 🎨 Анимации и эффекты

---

## 3.2 Где он используется на странице

### В отдельном файле (рекомендуется)

```html
<!-- Перед закрывающим </body> -->
<script src="script.js"></script>
```

### Внутри HTML

```html
<script>
  console.log("Hello, World!");
</script>
```

### Inline (не рекомендуется)

```html
<button onclick="alert('Привет!')">Нажми</button>
```

**Почему перед `</body>`?**
- ✅ HTML уже загружен (можно обращаться к элементам)
- ✅ Страница отображается быстрее

---

## 3.3 Базовый синтаксис

### Переменные

```javascript
// Объявление переменных
let name = "Сергей";
const age = 30;

// Вывод в консоль
console.log(name);
```

**Разница:**
- `let` — можно изменить: `name = "Олег"`
- `const` — константа: `age = 31` ❌ ошибка

### Функции

```javascript
// Объявление функции
function sayHello() {
  alert("Привет!");
}

// Вызов функции
sayHello();
```

**Функция с параметрами:**

```javascript
function greet(name) {
  return "Привет, " + name + "!";
}

let message = greet("Сергей");
console.log(message); // "Привет, Сергей!"
```

---

## 3.4 События (onclick и т.д.)

**События** = реакция на действия пользователя

### Базовый пример

```javascript
// Находим кнопку
let button = document.querySelector('button');

// Добавляем обработчик клика
button.addEventListener('click', function() {
  alert('Кнопка нажата!');
});
```

### Популярные события

```javascript
// Клик мышью
element.addEventListener('click', function() { ... });

// Наведение мышью
element.addEventListener('mouseover', function() { ... });

// Изменение в input
input.addEventListener('input', function() { ... });

// Отправка формы
form.addEventListener('submit', function() { ... });
```

### Пример: Показать/скрыть текст

```javascript
let button = document.querySelector('.toggle-btn');
let text = document.querySelector('.hidden-text');

button.addEventListener('click', function() {
  text.style.display = 'block';
});
```

**Что происходит:**
1. Находим кнопку с классом `.toggle-btn`
2. Находим текст с классом `.hidden-text`
3. При клике на кнопку → показываем текст

# 4. Как это работает в браузере

## 4.1 Что происходит когда открываешь сайт

### Пошаговый процесс

```
1. Ты вводишь URL → example.com
          ↓
2. Браузер запрашивает HTML файл с сервера
          ↓
3. Браузер читает HTML
          ↓
4. Находит <link> к CSS → загружает CSS
          ↓
5. Находит <script> к JS → загружает JS
          ↓
6. Строит DOM (дерево элементов)
          ↓
7. Применяет стили (CSSOM)
          ↓
8. Отрисовывает страницу (Rendering)
          ↓
9. Выполняет JavaScript
          ↓
10. Страница готова! 🎉
```

**Визуализация порядка:**

```
┌─────────────────────┐
│  index.html         │ ← Сначала HTML
├─────────────────────┤
│  styles.css         │ ← Потом CSS
├─────────────────────┤
│  script.js          │ ← Потом JS
└─────────────────────┘
```

---

## 4.2 HTML → CSS → JS (порядок загрузки)

### Критичность порядка

```html
<!DOCTYPE html>
<html>
<head>
  <!-- 1. CSS загружается ПЕРВЫМ -->
  <link rel="stylesheet" href="styles.css">
</head>
<body>
  <!-- 2. HTML рендерится -->
  <div class="container">
    <h1>Привет</h1>
  </div>
  
  <!-- 3. JS загружается ПОСЛЕДНИМ (перед </body>) -->
  <script src="script.js"></script>
</body>
</html>
```

**Почему такой порядок?**

```
❌ ПЛОХО:
JS загружается → пытается найти элементы → их ещё нет → ошибка

✅ ХОРОШО:
HTML загрузился → элементы есть → JS может с ними работать
```

Что показывает: Все запросы к серверу

- ✅ Проверить ошибки загрузки (404)

### 🔥 Главная функция для проверки AI-кода

**Inspect Element (Посмотреть код элемента):**

```
Правый клик на любом элементе → "Inspect" / "Посмотреть код"
```

**Что это даёт:**
- ✅ Видишь HTML элемента
- ✅ Видишь CSS стили
- ✅ Можешь менять и проверять

**Пример использования:**

```
AI сгенерировал кнопку, но она слишком маленькая

1. Правый клик на кнопку → Inspect
2. Видишь:
   <button class="btn">Нажми</button>
   
3. В стилях видишь:
   .btn {
     padding: 5px;  ← Слишком мало!
   }
   
4. Меняешь на: padding: 15px;
5. Кнопка стала нормального размера ✅
6. Копируешь изменение в свой CSS файл
```

# 5. AI и фронтенд-разработка

## 5.1 Что AI умеет генерировать

### Сильные стороны AI

**✅ AI хорошо генерирует:**

1. **HTML структуру**
   - Семантичная разметка
   - Правильная иерархия заголовков
   - Формы с валидацией

2. **CSS стили**
   - Flexbox/Grid layouts
   - Адаптивный дизайн
   - Анимации и переходы

3. **Базовый JavaScript**
   - Обработчики событий
   - DOM манипуляции
   - Простые функции

### Слабые стороны AI

**❌ AI плохо справляется с:**

1. **Кроссбраузерной совместимостью**
   - Может сгенерировать код, который не работает в Safari
   - Забывает вендорные префиксы

2. **Производительностью**
   - Может создать медленный код
   - Не оптимизирует картинки

3. **Доступностью (a11y)**
   - Забывает `alt` для картинок
   - Не добавляет ARIA-атрибуты

4. **Сложной логикой**
   - Многоуровневая вложенность
   - Состояния приложения
